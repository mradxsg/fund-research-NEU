# 智能投顾系统—基金研究子系统 系统设计说明书

## 1. 系统架构
本系统采用前后端分离的 B/S 架构。

**前端**：Vue3 + Vite + Element Plus + ECharts 构建的单页应用（SPA），通过 Axios 调用后端 RESTful API。

**后端**：Spring Boot 3.2.5 + MyBatis-Plus 3.5.7，分层为 Controller → Service → Mapper → 数据库。

**数据库**：MySQL 8.0（开发环境），已适配金仓 KingbaseES V9（兼容 MySQL 模式）。

**AI 服务**：通过后端代理转发至本地 Ollama 运行的 DeepSeek-R1:1.5b 模型。

**鸿蒙端**：独立元服务项目，调用后端 `/api/funds/code/{code}` 获取净值显示在万能卡片上。


技术架构图：
```
[前端 Vue3]  ←→  [Spring Boot 3]  ←→  [MySQL / 金仓]
                  ↕
               [Ollama + DeepSeek]
```

## 2. 功能模块分解

| 模块编号 | 模块名称 | 功能简述 |
|----------|----------|----------|
| M1 | 基金查询 | 关键字搜索、标签筛选（同维或/跨维且）、分页、保存组合 |
| M2 | 基金公司查询 | 公司搜索、标签筛选、查看旗下基金、保存组合 |
| M3 | 基金经理查询 | 经理搜索、标签筛选、查看管理基金、保存组合 |
| M4 | 基金画像 | 基本信息、净值走势图、持仓分析、业绩归因、公告 |
| M5 | 组合管理 | 组合列表、详情、删除、批量操作 |
| M6 | 标签体系 | 提供基金/公司/经理三维度标签树接口 |
| M7 | 用户模块 | 注册、登录、个人空间（改昵称、改密码） |

附加功能：AI 智能问答（A1）、鸿蒙元服务接口（A2）

## 3. 数据库设计

数据库共 12 张核心表：
- `user`（用户）
- `fund_company`（基金公司）
- `fund_manager`（基金经理）
- `tag`（标签，支持树形层级）
- `fund_info`（基金基本信息）
- `fund_nav`（净值历史）
- `fund_sector`（行业配置）
- `fund_stock`（重仓股）
- `fund_tag`（基金-标签关联）
- `fund_announcement`（公告）
- `portfolio`（组合主表）
- `portfolio_item`（组合明细）

实体关系简述：一个公司有多个基金，一个经理管理多个基金，基金与标签多对多，用户与组合一对多，组合与基金多对多。

## 4. 核心模块设计

### 4.1 基金查询模块
- FundController、FundService、FundServiceImpl、FundMapper、FundTagRelationMapper 协同工作。
- 标签筛选逻辑：同维度标签使用 IN 子查询实现 OR，跨维度使用自定义 SQL 实现 AND。

### 4.2 基金画像模块
- FundController.getFundProfile() → FundService.getFundProfile()
- 聚合查询基金基本信息、公司、经理、标签、净值、持仓、公告。

### 4.3 用户模块
- UserController、UserService、UserMapper 实现注册、登录、个人信息维护。

### 4.4 组合管理模块
- PortfolioController、FundService、PortfolioMapper、PortfolioItemMapper 协同完成增删查。

### 4.5 标签树模块
- TagController、FundService.getTagTree() 根据维度查询标签并递归构建树形结构，返回 TagTreeNodeDTO。
