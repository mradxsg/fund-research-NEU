# 智能投顾系统 — 基金研究子系统

## 项目简介
本项目是智能投顾系统的基金研究子系统Web端，基于**Spring Boot + Vue3** 的前后端分离架构，面向全市场公募基金提供查询、筛选、画像展示和组合管理功能。系统集成**本地大语言模型（DeepSeek-R1）**实现智能问答，适配**鸿蒙元服务**和**国产金仓数据库**，并完成了单元测试与性能测试。

## 功能说明
- **基金查询**：支持按基金代码/名称模糊搜索，结合多级标签树（基金类型、风险等级、业绩评级）进行多维度筛选（同维度标签OR，跨维度标签AND），分页展示基金列表。
- **基金公司/经理查询**：支持按公司名称或经理姓名搜索，通过标签筛选，查看旗下/管理的基金，并可将结果保存为组合。
- **基金画像**：展示基金基本信息、净值走势折线图、行业配置饼图、前十大重仓股表格、业绩归因文本、最近5条公告。
- **组合管理**：用户登录后可创建、查看、删除自定义基金组合，支持批量操作。
- **用户系统**：注册、登录、个人空间（昵称修改、密码修改）。
- **AI 智能问答**：页面右下角悬浮机器人按钮，点击唤起聊天窗口，基于本地部署的DeepSeek-R1:1.5b模型回答基金投资相关问题。
- **鸿蒙元服务**：提供独立鸿蒙应用代码（`harmonyos/`），通过调用后端接口实现基金净值查询与万能卡片展示。

## 技术栈
| 层级 | 技术 |
|------|------|
| 后端 | Spring Boot 3.2.5、MyBatis-Plus 3.5.7、MySQL 8.0 |

## 安装部署指南

### 环境要求
- **JDK**：17 及以上
- **Node.js**：18 及以上
- **MySQL**：8.0（或 KingbaseES V9）
- **Maven**：3.8 及以上
-  Ollama（用于AI对话功能）

### 数据库初始化
1. 创建 MySQL 数据库 `fund_db`。
2. 执行 `docs/fund_db.sql` 创建12张业务表，导入模拟数据（50只基金、13000条净值、标签、持仓等）。

### 后端启动
```bash
# 进入项目根目录（包含 pom.xml）
mvn clean package
java -jar target/fund-research-0.0.1-SNAPSHOT.jar
```
也可直接在 IDE（STS4 / IntelliJ IDEA）中运行 `FundResearchApplication.java`，默认端口 `8080`。
（注：直接以java项目或SpringBoot项目运行，可能因环境差异出现异常，可以以Maven Build…方式进行运行。）

### 前端启动
```bash
cd frontend
npm install
npm run dev
```
浏览器访问 `http://localhost:5173` 即可使用。

### AI 功能启用
- 安装 [Ollama](https://ollama.com/download)
- 拉取模型：`ollama run deepseek-r1:1.5b`
- 启动服务：`ollama serve`
- 后端默认连接 `http://localhost:11434`，无需额外配置。

### 鸿蒙元服务运行
- 使用 **DevEco Studio** 打开 `harmonyos/` 目录。
- 根据需要修改 `entry/src/main/ets/common/constants/Constants.ets` 中的后端接口地址。
- 连接模拟器或真机，点击 **Run** 编译运行。

## 使用说明
1. 打开前端页面，点击右上角「登录」，使用测试账号 `test01 / 999999` 登录（或自行注册）。
2. 在「基金」页面，通过搜索框输入关键字或勾选左侧标签树筛选基金，点击「详情」查看基金画像。
3. 勾选感兴趣的基金，点击「保存为组合」，输入名称即可将筛选结果保存为自定义组合。
4. 在「我的组合」页面可以查看、删除已保存的组合，点击「批量操作」可以多选删除。
5. 右下角 🤖 按钮可唤起 AI 智能问答，输入基金相关问题即可获得回答。
6. 「公司」和「经理」页面支持按名称搜索，并可查看旗下/管理的基金列表。
